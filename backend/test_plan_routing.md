# 对话路由修复 — 测试计划

## 问题描述

顾客在对话中表达具体产品需求（如"推荐罗汉果茶"）时，系统将用户拖入体质测试流程，即使顾客明确拒绝也继续追问体质问题。constitution 阶段没有退出机制。

## 测试场景

### A. 基础产品搜索（greeting 阶段）

| ID | 场景 | 用户输入 | 期望行为 |
|---|---|---|---|
| A1 | 从欢迎页直接问产品 | "有没有罗汉果茶" | 返回罗汉果茶相关产品推荐，不进入筛查/体质 |
| A2 | 从欢迎页问价格 | "罗汉果茶多少钱" | 返回价格信息，附产品推荐 |
| A3 | 从欢迎页模糊要推荐 | "推荐几个适合我的" | 引导进入体质了解（没有具体产品名，合理） |

### B. 对话中途转产品搜索

| ID | 场景 | 用户输入 | 期望行为 |
|---|---|---|---|
| B1 | 筛查阶段问产品 | 筛查时说"不要筛查，给我推荐罗汉果茶" | 跳过筛查，直接推荐罗汉果茶产品 |
| B2 | 体质提取阶段问产品 | 体质闲聊时说"算了，你就给我罗汉果茶吧" | 退出体质流程，直接搜索罗汉果茶 |
| B3 | 自适应追问阶段问产品 | 追问时说"不用，直接推罗汉果茶" | 退出追问，直接搜索罗汉果茶 |
| B4 | 场景阶段问产品 | 说困扰时说"还是推荐罗汉果茶吧" | 搜索并推荐罗汉果茶 |

### C. 产品搜索持续对话

| ID | 场景 | 用户输入 | 期望行为 |
|---|---|---|---|
| C1 | 推荐后用模糊词要更多 | "再推荐几个" | 基于前文"罗汉果茶"继续推荐相关产品 |
| C2 | 推荐后换产品 | "有没有山楂相关的" | 搜索山楂相关产品 |
| C3 | 推荐后回体质流程 | "帮我看看体质吧" | 正常进入体质流程 |

### D. quick_replies 一致性

| ID | 场景 | 触发条件 | 期望行为 |
|---|---|---|---|
| D1 | 产品推荐后的按钮 | 系统返回产品推荐 | 显示"推荐更多产品/重新了解体质/看看产品目录" |
| D2 | 目录展示后的按钮 | 系统返回产品目录 | 显示"饼干/面包/茶/香囊/帮我看看体质" |

## 预期修复点

1. **`_handle_constitution_extract`**: 收到用户消息后，先检查是否含产品搜索意图，是则设置 intent 并跳过提取
2. **`_handle_constitution_adaptive`**: 收到用户消息后，先检查是否含产品搜索意图，是则退出 adaptive 循环
3. **`_search_products`**: 当当前消息搜索无结果时，尝试从 conversation history / state 中提取之前提到的产品名
4. **所有 stage handler**: 统一处理 `search_product` intent 的退出逻辑
5. **`_handle_scene`**: 恢复 intent 透传（当前硬编码为 `continue_flow`）

## 执行方式

Python 脚本 `backend/test_routing.py`，通过 HTTP 调用本地 API，逐场景验证。输出通过/失败和具体差异。

## 验收标准

- A1-A3 全部通过
- B1-B4 全部通过（核心：constitution 阶段可被产品搜索中断）
- C1-C3 全部通过
- D1-D2 全部通过
