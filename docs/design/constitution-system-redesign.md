## 3. 核心设计：两阶段快速定位

### 问题：逐个问 10 种体质太慢

传统方式需问 5-10 个问题才能锁定体质。顾客耐心只有 3-4 轮。

### 方案：开门见山 + 二选一分叉

**阶段一：开放式自由描述（1 轮）**

顾客说身体感受 → AI 提取信号，缩小到 2-3 个候选体质。

示例：
```
AI: "最近身体感觉怎么样？随便说说～"
顾客: "最近总觉得累，消化不好"
→ AI 内部判断：候选 [气虚质, 痰湿质]，概率 80%
```

**阶段二：鉴别追问（最多 2 轮）**

在候选体质间做鉴别，不问无关问题。

示例：
```
AI: "了解了～再问你一个：你平时是怕冷多一点，还是觉得身体沉、肚子胀比较多？"
     ↑ 鉴别气虚质(怕冷倾向) vs 痰湿质(体沉腹胀)
顾客: "身体沉，肚子胀"
→ AI 锁定：痰湿质（概率 90%），进入困扰确认
```

**关键原则：**
- 最多问 3 个问题（含开放描述）
- 每个问题都在缩小候选范围，不是逐项排查
- 顾客说"不知道/都还好"→ 直接归为平和质，走通用推荐

### 对话流程（优化后）

```
greeting（1轮）
  ↓
开放描述（1轮）:"最近身体感觉怎么样？"
  ↓
鉴别追问（0-2轮）:在2-3个候选体质间快速鉴别
  ↓
体质确认（1轮）:"你偏XX体质，我帮你搭配几款～"
  ↓
困扰确认 + 推荐（1轮）
────────────────────
总计：3-5 轮即可完成推荐
```

## 4. 套餐设计：2+2+1 模式

每个体质套餐包含 **5 件产品**，结构：

```
核心产品 × 2：直接针对该体质（必选）
搭配产品 × 2：与该体质相关或互补（增强客单价）
锦上添花 × 1：应季/香囊/礼盒（提升体验）
```

### 示例：气虚质套餐

| 角色 | 产品 | 理由 |
|------|------|------|
| 核心 | 百合山药吐司 | 山药补气健脾 |
| 核心 | 黄芪茯苓茶 | 黄芪+茯苓经典补气组合 |
| 搭配 | 山药紫薯饼干 | 山药同源，零食化补充 |
| 搭配 | 红枣核桃酥 | 气血双补，口感好 |
| 锦上添花 | 安神香囊 | 气虚多伴睡眠差，香囊助眠 |

### 货架逻辑

每个体质的套餐在后台是一个 **推荐池（pool）** 而非固定清单：
- 必选 2 件 + 推荐 5-8 件供店员灵活搭配
- 对话时 AI 从池中根据顾客困扰二次筛选
- 后台可随时增删池中产品

## 7. 风险与对策

| 风险 | 对策 |
|------|------|
| 10 种体质让顾客选择困难 | 顾客不需要选择——AI 自动判断，顾客只需回答 1-3 个问题 |
| 鉴别追问可能不准 | 允许店员在后台修正体质标签，后续对话基于修正后的体质 |
| 产品池太大店员维护困难 | 提供默认池，店员可以不改，也可以基于默认池增减 |
