# 焙草集语音输入功能设计

> 日期：2026-05-18
> 目标：为聊天页面增加按住说话语音输入，替代打字

---

## 1. 目标

让顾客在手机端通过语音快速输入症状，替代打字，提升对话效率。

---

## 2. 界面布局

```
┌──────────────────────────────────┐
│         焙草集 AI 健康顾问              │  ← header
├──────────────────────────────────┤
│                                  │
│     [消息列表 scroll-view]       │
│                                  │
│              ↓ 60s倒计时          │
├──────────────────────────────────┤
│ [🎤] [输入框...        ] [发送]  │  ← bottom-bar
└──────────────────────────────────┘
```

- 麦克风按钮放置在输入框左侧（打字习惯保留）
- 输入框和发送按钮保留在右侧
- "正在聆听..." 提示显示在麦克风和输入框之间

---

## 3. 交互流程

### 3.1 完整录音交互

1. **按住麦克风按钮**
   - 按钮变红 + 倒计时从 60s 递减
   - "正在聆听..." 提示显示在麦克风和输入框之间
   - 录音开始

2. **用户正常松手**
   - 倒计时结束或用户主动松手
   - 显示"识别中..."提示
   - 微信返回识别文字，自动发送到对话

3. **用户向上滑动并松手（取消）**
   - 录音取消，按钮恢复
   - 无任何提示，可再次按住说话

4. **识别失败**
   - 输入框下方短暂显示红色提示"识别失败，请重试"
   - 用户可重新按住说话

---

## 4. 状态枚举

| 状态 | 描述 |
|------|------|
| `idle` | 初始状态，麦克风可按 |
| `recording` | 录音中，倒计时进行中 |
| `recognizing` | 识别中，等待文字返回 |
| `error` | 识别失败，显示错误提示 |

---

## 5. 详细规则

### 5.1 录音时长
- 最长 60 秒（与微信语音消息上限一致）
- 到达 60 秒时强制停止录音并发送
- 录音时长 < 1 秒时，当作"没说话"，显示错误提示

### 5.2 麦克风按钮样式（录音状态）
- 背景色：`#8B2500`（深红，与处方卡印章色一致）
- 倒计时数字显示在按钮内
- 手指滑向上方时，按钮上方显示"松开取消"提示

### 5.3 并发处理
- 录音进行中收到 bot 回复 → 正常追加消息列表，录音不中断
- 录音进行中点击购物车 → 允许，录音不中断

### 5.4 识别结果
- 直接发送，不进入输入框，用户不可编辑
- 与文字输入流程一致，仅输入方式不同

---

## 6. 技术方案

### 6.1 前端实现
- 使用 `wx.createRecordManager` 管理录音
- 微信内置语音识别（`wx.startRecord` 结束后自动返回识别文字）
- 录音文件临时存储，识别完成后丢弃

### 6.2 语音识别来源
- 微信内置语音识别（自动模式）
- 不依赖后端 ASR 服务

### 6.3 API 接口
- 现有 `/api/chat/send` 接口不变，接收文本
- 语音流程：录音 → 微信转文字 → 文本发送给后端
