# 焙草集AI销售助手系统——设计文档

## 背景

焙草集是一家开在医院里的药食同源门店，销售饼干、面包、茶饮、香囊等产品（约100个SKU，4个品类）。顾客进店流失率高，店员专业培训成本高。需要一个AI工具辅助销售，提升进店转化率和客单价。

## 系统概述

微信小程序为载体，AI对话式引导顾客完成体质倾向辨识和场景收集，推荐适合的商品套餐。**购买环节对接有赞**——有赞承载商品管理、订单、支付、物流等电商底座，本系统专注AI推荐和顾客健康数据管理。

## 与有赞的关系

本系统 = AI推荐层，有赞 = 电商底座。职责边界：

- **我们做**：AI对话、体质分析、商品推荐、健康报告、顾客数据沉淀
- **有赞做**：商品主数据管理、库存、订单、支付、物流、售后

对接方式：商品从有赞同步到本系统（保留有赞商品ID作为关联键）；推荐结果通过有赞小程序SDK跳转下单。

## 三端架构

| 端 | 用户 | 载体 | 核心功能 |
|---|---|---|---|
| C端 | 顾客 | 微信小程序 | 扫码或群链接进入 → 安全筛查 → 对话式体质辨识 → 场景收集 → 接收推荐套餐 → 查看健康报告 → 点击商品跳有赞下单或到店购买 |
| 员工端 | 店员 | 微信小程序（店员入口） | 查看顾客列表及对话记录、推荐结果、顾客报告；可补充备注；可替顾客发起对话 |
| 后台端 | 店长/运营 | PC网页 | 数据统计（体质分布、热门场景、转化率）、商品标签维护（体质/场景维度）、有赞商品关联管理、顾客数据查询导出 |

## C端对话流程

### 阶段1：开场破冰

AI打招呼，告知大约耗时，建立轻松预期。

### 阶段2：前置安全筛查

免责声明后，询问顾客是否属于以下群体（可多选）：

- 怀孕或备孕中
- 哺乳期
- 确诊重大疾病（如肿瘤、严重肝肾疾病等）
- 正在服用处方药
- 以上都没有

分流逻辑：
- 选前三项任一 → 温和拦截：建议咨询医生后再使用产品，仅可浏览商品目录，不做推荐
- 仅选"服用处方药" → 追问药物类别以做冲突排除，不愿透露则同上拦截
- "以上都没有" → 继续

### 阶段3：顾客信息采集

询问顾客称呼和手机号（选填），说明用于生成调理报告。不愿留信息仍可继续看推荐，但不生成报告。

### 阶段4：体质辨识（对话式，约6轮）

每轮一问，用自然语言而非表单。只问顾客主观能感受到的，不问需专业知识判断的（如舌苔、脉象）。示例：

- 冬天手脚偏凉还是偏暖？
- 容易上火、口干、长痘吗？
- 容易疲惫、说话没力气吗？
- 饭后容易腹胀、大便偏黏吗？
- 比别人怕冷还是怕热？

### 阶段5：场景收集（1-2轮）

- "最近有没有什么特别困扰你的？"（给出方向提示：睡不好、消化差、容易累、皮肤差）
- 追问细节（如"睡不好是睡不着还是半夜容易醒？"）

### 阶段6：推荐结果

综合体质+场景，推荐1个套餐（2-3款商品，跨品类组合），附带每款商品的成分解释和匹配理由，展示套餐价及省金额。提供"微信下单"和"到店自取"两个按钮。

### 阶段7：健康报告

生成一份精美报告（小程序Canvas长图渲染），内容结构：

1. 体质倾向概述（只说"偏XX体质"，杜绝诊断用词）
2. 日常食养建议
3. 推荐商品套餐及成分说明
4. 生活小贴士（作息、运动、情绪）
5. 重要提示（固定文案："本报告基于传统食养理念，仅为调理建议参考，不构成医疗诊断。如有健康问题请咨询执业医师。"）

每页底部固定注明：本推荐不构成医疗建议。

## 推荐引擎

### 匹配逻辑

两层匹配：

1. **场景匹配（主维度）**：顾客描述的困扰 → 匹配商品库"核心场景标签"
2. **体质精排（安全+增效）**：
   - 排除：禁忌标签命中该体质的商品
   - 优选：成分适合该体质的加权
   - 互补：跨品类且功效互补的商品形成天然套餐

### 套餐组合规则

- 每个套餐2-3款商品，来自不同品类（饼干+茶，面包+茶+香囊等）
- 不推两个同类产品进同一套餐
- 单品直减到套餐价，展示节省金额

## 技术方案

### 对话引擎

状态机 + LLM：

- 状态机控制阶段流转和问题边界（保证不跑偏）
- LLM负责自然语言生成和顾客自由文本理解
- 非全开放聊天，token消耗可控

### 系统架构

```
[微信小程序]
    ├── AI对话 → [后端服务]
    │               ├── 对话管理器（状态机 + LLM调度）
    │               ├── 推荐引擎（体质+场景 → 商品/套餐匹配）
    │               └── 商品知识库（成分、功效、禁忌、场景标签）
    │
    └── 购买 → [有赞小程序SDK]
                   └── 有赞云（订单/支付/物流/库存）
```

### 有赞对接

- **商品同步**：有赞商品上架时，在本系统后台关联有赞商品ID，补充体质/场景/禁忌标签
- **下单跳转**：推荐结果中每个商品挂有赞商品链接，顾客点击跳有赞小程序完成购买
- **到店购买**：顾客出示推荐结果，店员在有赞POS或扫码完成线下成交

### 数据存储

- 顾客对话记录、体质判断结果、推荐历史绑定微信用户
- 支撑后续复购推荐和精准运营

## 防诊断红线

1. AI话术只描述成分和传统调理作用，不说"治疗""治愈"
2. 体质只说"偏颇倾向"，不说"病证""诊断"
3. 所有报告页和对话底部标注"不构成医疗建议"
4. 安全筛查前置拦截高危人群

## 商品数据现状

4个Excel文件（biscuit、bread、tea、toy），每个约12-32条，共约100条SKU。每条包含：

- SKU_ID、产品特色、产品名称
- 核心成分、核心场景标签（面向顾客）
- 销售话术（面向店员）
- 禁忌标签

数据需导入数据库并可能补全体质关联维度。
